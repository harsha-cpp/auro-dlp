# AURO-DLP — Security & Tamper-Resistance

## 1. Trust Boundaries

```
[ User ] -- (untrusted, can browse) --> [ Browser ]
[ Browser ] -- mTLS loopback --> [ Agent (LocalSystem) ]
[ Agent ] -- mTLS WAN --> [ Policy Server ]
[ Policy Server ] -- signed bundle --> [ Agent ]
```

The user is **untrusted with respect to bypass attempts**. The browser is trusted only for its DOM-isolated extension code (force-installed). The agent runs at higher privilege than the user.

## 2. Bypass Prevention Matrix

| Bypass attempt | Defense |
|---|---|
| Disable extension | Force-installed via GPO; `ExtensionInstallBlocklist=*` blocks user-installed alternatives; agent flags missing-extension heartbeat. |
| Use Firefox / portable browser | AppLocker/WDAC denies; agent log-only mode for unmanaged browsers; periodic audit. |
| Kill `auro-agent.exe` | Service runs as LocalSystem; user lacks `SERVICE_STOP`; watchdog process restarts; dashboard alerts on missed heartbeat. |
| Edit policy YAML on disk | YAML is Ed25519-signed; agent rejects unsigned/changed files; falls back to last good signed copy. |
| Replay-attack the agent API | mTLS with per-install cert; nonce-based request signing in inspect requests; loopback only. |
| Spoof Gmail compose URL | Extension manifest restricts to `https://mail.google.com/*`; content script verifies `window.top === window.self` and `location.origin`. |
| Embed PHI in image to evade text scan | OCR pipeline (eng+hin) on every uploaded image; refused if OCR fails and policy is `strict`. |
| Embed PHI in password-protected ZIP | Agent treats password-protected archives as `risk=high`; policy can WARN/BLOCK without inspection. |
| Steganography | Out of scope; flagged in roadmap. |
| Screenshot the screen | Foreground-process monitor for `SnippingTool.exe`, `ScreenSketch.exe`, `mspaint.exe` post-PrtSc; logged & optionally warned. |
| Copy patient data to another app then send | Out of scope for this DLP — pair with Microsoft Purview / Symantec EPC clipboard control. |

## 3. Cryptography

| Use | Algorithm |
|---|---|
| Loopback transport | TLS 1.3, ECDHE + AES-256-GCM, mTLS |
| WAN transport | TLS 1.3 + cert-pinning to policy-server CA |
| Policy bundle signing | Ed25519 (per RFC 8032) |
| Local audit at rest | AES-256-GCM, key derived via DPAPI tied to LocalSystem |
| Override token | TOTP HMAC-SHA-256, 30-s window, single-use, signed by server |

## 4. Code Integrity

- Agent: built with `-trimpath -ldflags="-s -w -buildid="`, signed with EV cert, embedded SHA-256 fingerprint of itself, self-check on launch.
- Policy server: SBOM generated by `npm ls --all --json`.
- Extension: pinned content hash in GPO; Chrome enforces.

## 5. Audit Log Schema

Append-only NDJSON, AES-GCM frames:

```json
{
  "ts": "2026-05-07T11:34:21.044Z",
  "endpoint": "WS-CARDIO-04",
  "user": "dr.suresh@hospital.in",
  "incident_id": "01HXAB...",
  "verdict": "BLOCK",
  "rule_ids": ["IN.AADHAAR","MED.MRN","MED.ICD10"],
  "match_counts": [2,1,1],
  "attachment_hashes": ["sha256:..."],
  "recipient_domain_hashes": ["sha256:..."],
  "policy_version": "2026-05-01-r3",
  "signature": "ed25519:..."

```

Each line is signed; tampering invalidates the chain (signature uses prior-record hash, Merkle-style).

## 7. SIEM Integration

`policy-server/src/services/siem.js`:

- Splunk HEC (token in `SIEM_HEC_TOKEN`).
- Syslog RFC 5424 over TLS.
- Generic webhook (HMAC-SHA-256 signed).

Events emitted: `incident.created`, `incident.override`, `agent.tamper_detected`, `policy.updated`, `auth.failure`.

## 8. Incident Response Runbook

1. Dashboard alert → security on-call paged via SIEM webhook.
2. On-call opens incident, reviews masked snippet (reveal-token workflow if raw text is needed).
3. If user invoked override, on-call confirms business reason. If not, contact user manager.
4. Escalate to DPO if Aadhaar / health-record exfiltration confirmed.
5. Roll out tightened policy if pattern recurs.

## 9. Penetration Test Checklist (pre-prod)

- [ ] Agent listens only on 127.0.0.1.
- [ ] mTLS rejects connections without client cert.
- [ ] Policy bundle with mutated signature is rejected.
- [ ] User cannot stop `AURO-DLPSvc`.
- [ ] Watchdog restarts agent after `taskkill /F`.
- [ ] Extension blocked from running on `mail.google.com/u/X` non-managed accounts (optional, policy choice).
- [ ] Audit log signature chain is valid after 24 h soak.
- [ ] Update bundle with mutated hash is rejected.
